What is Claude Code?

Claude Code is a powerful AI-driven development environment that accelerates your workflow by automating repetitive tasks, analyzing code, and executing complex operations with precision. It combines AI intelligence with real-time feedback to help you ship faster.

🧠 不只是「聊天」——這是一種開發方法論

AI 輔助開發的
真實樣貌

Vibe Coding 不是把需求丟給 ChatGPT 複製貼上。
它是一個持續的開發迴圈——AI 理解你的程式架構、讀取錯誤訊息、跨檔案重構，更像是一位隨時在線的 pair programmer。

往下滑
認識概念

Vibe Coding ≠ 用 AI 聊天

一般人對 AI 寫程式最大的誤解：以為 Vibe Coding 就是把問題貼給 ChatGPT。這兩件事在技術本質上截然不同。

⚙️ 核心技術機制

ChatGPT 和 Claude 是通用型 LLM 對話介面，沒有 codebase 索引——你需要手動貼程式碼。但它們擁有最大的 context window（Claude 200k tokens）和最強的推理能力。Claude 在程式碼理解、長文件分析和邏輯推理上特別強。ChatGPT 在廣度和工具整合（Code Interpreter、DALL-E）上有優勢。兩者都支援專案功能（Projects/Memories）讓 context 跨對話保留。

🔄 最適合的工作流程

作為 Cursor/Windsurf 的補充工具，而非替代。最佳使用方式：① 架構設計討論（"我要做 XXX，哪種架構最合適？"）；② 大型單一檔案分析（把整個 1000 行檔案貼給 Claude 審查）；③ 技術選型諮詢；④ 給 Cursor 的複雜 prompt 先用 Claude 打草稿。

📬 實際 Prompt 範例
📋 架構討論 Prompt（給 Claude）
"我在做一個 B2B SaaS，預計 100 個企業客戶，每個客戶有獨立的資料隔離需求。幫我分析 multi-tenant 架構的三個選項（shared schema / separate schema / separate database）的 trade-off，考慮我的技術棧是 Next.js + PostgreSQL + Prisma，給出具體建議。"
⚔️ vs Cursor
ChatGPT/Claude 勝：不需要 IDE；任何裝置即用；超長 context window；最強推理能力；對話更自然靈活；適合非結構化的探索
Cursor 勝：真實 codebase 感知；自動修改檔案；Terminal 整合；適合連續開發迭代而非一次性問答
🏢 企業 vs 個人 Verdict
✅ 所有開發者的補充工具

ChatGPT/Claude 不是 Vibe Coding 的主力工具，但是最好的「副駕駛」。建議工作流：Cursor 做日常開發，Claude 做架構討論和大局分析。企業版（Claude for Teams / ChatGPT Enterprise）有更好的隱私保護。

⚠️ 限制與缺點
  • 沒有 codebase 索引，每次需要手動貼相關程式碼
  • 無法直接修改檔案，所有輸出都需要複製貼上
  • 不適合連續開發迭代（每次對話缺乏上下文連貫性）
  • 免費版有訊息限制和慢速回應

層次之別

Vibe Coding vs AI Agent
你是指揮家，它是全能義肢

Vibe Coding 是大腦的「指揮藝術」，而 AI Agent 是實現藝術的「全能義肢」。兩者不是競爭關係，而是協作層次不同——搞清楚這個，你才能同時駕馭兩種力量

常見問題

你可能想知道的事

Vibe Coding 和普通 Prompt 對話差在哪？一句話說明。+
普通 Prompt 是無狀態的一次性問答，Vibe Coding 是有狀態的持續開發迴圈，AI 索引了你的 codebase、能讀取執行錯誤、能跨檔案修改，更像是一位了解你整個專案的 pair programmer。
不懂程式的人可以學 Vibe Coding 嗎？+
可以，但有一個誠實的限制：你越懂程式，Vibe Coding 的效果越好。完全不懂程式的人可以做出簡單的工具，但遇到複雜問題時可能難以判斷 AI 的輸出是否正確。建議同步學習基礎的 HTML/CSS/JavaScript 概念，即使只是閱讀層面的理解，也能讓你的 Vibe Coding 品質大幅提升。
AI 有時候改了一個地方，另一個地方就壞掉，怎麼辦？+
這是 Vibe Coding 中最常見的問題，叫做「跨檔案一致性問題」。解法是：① 在 prompt 中加入「確認這個修改不會影響其他呼叫這個函式的地方」；② 要求 AI 在修改後列出「受影響的檔案清單」；③ 養成寫測試的習慣，讓 AI 改完後跑測試確認沒有破壞其他功能。
Cursor 和 Claude 在使用上有什麼本質差異？+
Cursor 是整合在編輯器裡的開發環境，它有完整的 codebase 索引、Terminal 整合和跨檔案感知——這是真正的 Vibe Coding 體驗。Claude 是網頁對話介面，沒有 codebase 索引，你需要手動貼程式碼，但它的 200k token context window 很長，適合分析大型單一檔案或做架構討論。兩者可以配合使用：Cursor 用來日常開發，Claude 用來做大局分析和架構設計。
AI 生成的程式碼有安全問題嗎？+
確實存在安全盲點。AI 優化的是「功能能跑」，不一定會主動考慮 SQL Injection 防護、XSS、API key 洩漏等安全問題。解法：① 涉及用戶資料的功能，明確要求「做完後幫我做安全審查」；② 永遠不要把 API key 或密碼直接寫在程式碼裡，要用環境變數；③ 生產環境的程式碼，找有安全背景的人做 code review。
Vibe Coding 做出來的東西，能到什麼規模？+
個人工具、Landing Page、MVP 產品、中型 SaaS——都有成功案例。目前的限制主要在於：超過幾萬行的大型 codebase 管理困難（context window 不夠放），以及複雜的分散式系統架構還是需要有經驗的工程師設計。但對 90% 的個人專案和小團隊需求來說，Vibe Coding 已經相當夠用，而且速度遠比傳統開發快。
